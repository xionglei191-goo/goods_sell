## 2. 色彩系统

### 管理后台色彩

定位：Sales Intelligence Dashboard / Data-Dense Dashboard。后台优先服务高频运营、销售、库存和财务判断，采用浅底数据密集布局，橙色表达交易和主操作，绿色表达增长/完成，红色表达风险/逾期/库存预警。

| 增长色(Growth) | 成功绿 | `#16a34a` |

## 8. 组件规范

### 数据表格
- 支持列排序、筛选
- 行悬浮高亮
- 支持分页(默认20条/页)
- 操作按钮在最右列
- 移动端转为卡片列表

### 表单
- 标签在上方(Label on top)
- 必填字段带红色星号
- 实时验证+错误提示
- 操作按钮固定在底部

### 按钮层级
- Primary: 实心主色按钮（页面主操作）
- Secondary: 描边按钮（次要操作）
- Ghost: 文字按钮（更多操作）
- Danger: 红色按钮（删除/取消）

### 数据卡片(Dashboard)
- 暖白卡片+暖米细边框+轻微阴影
- 图标+数字+趋势箭头
- 交易指标使用橙色，增长/完成使用绿色，风险/逾期/预警使用红色
- 页面大背景使用接近白色的暖奶油底；卡片与背景只保留轻微层级差。陶土红只用于选中态、主操作、交易图表和关键强调，不做整页铺底
- Hover 使用边框/背景变化，不使用会造成布局位移的缩放动画

### 商城 Block 卡片
- 产品、分类和活动入口使用暖色块卡，圆角 `8px`
- 分类色彩：酒类品牌红、食品促销橙、饮料成功绿
- 促销标签和 CTA 可使用橙色；保障、配送、完成态使用绿色
- 移动端保持 2 列产品网格，横向推荐卡片固定宽度，避免滑动时布局跳动

### 语义类使用
- 商城卡片统一使用 `shop-block-card`，商城输入框使用 `shop-form-input`，空状态使用 `shop-empty-state`。
- 商品图和无图占位使用 `ProductArt` 的暖色占位规则，不在业务页手写灰色“暂无图片”块。
- 后台页面容器使用 `surface-panel` / `dashboard-data-card`，表单使用 `form-input`，表格行使用 `dashboard-table-row`。
- 经销商端卡片使用 `dealer-card`，空状态使用 `dealer-empty-state`，状态标签使用 `dealer-status-badge`。
- 禁止新增大面积 `bg-white`、`bg-slate-*`、`border-neutral-200` 和蓝色 focus；透明高光、日志代码块和全局兜底规则除外。

### AI 悬浮气泡
- 固定在页面右下角，商城、后台和经销商端保持一致入口。
- 展开后采用对话面板，不遮挡主流程关键按钮；移动端宽度适配小屏。
- 查询结果使用摘要卡，展示标题、摘要、关键字段和可跳转链接。
- 写操作使用确认卡，明确展示操作对象、金额/数量、风险等级和确认按钮。
- 高风险操作必须展示二次确认输入框，按钮文案避免模糊表达。
- 错误状态要区分未登录、无权限、参数缺失、执行失败和模型不可用。

## 9. 动效规范

```css
--transition-fast: 150ms ease;    /* 按钮hover */
--transition-base: 200ms ease;    /* 一般过渡 */
--transition-slow: 300ms ease;    /* 弹窗/抽屉 */
--transition-spring: 300ms cubic-bezier(0.34, 1.56, 0.64, 1); /* 弹性效果 */
```

## 10. 适老化设计(可选模式)

- 大字体模式(字号增大25%)
- 高对比度配色
- 大触控区域(最小48px)
- 简化操作流程
- AI 气泡快捷入口，减少多级菜单查找
- 语音输入搜索/AI 输入预留入口(后续)
