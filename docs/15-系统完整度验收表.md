# 华启商城系统完整度验收表

本表用于验证系统本体是否做完，不检查域名、微信真实密钥、支付商户、税控和酒类资质。`npm run check:system` 只是自动门禁，最终结论必须结合本表的真实入口、真实操作、权限、审计、异常处理和证据。

## 1. 验收口径

| 维度 | 验收标准 |
|------|----------|
| 真实入口 | 用户能从导航、页面入口或 AI 能力目录找到功能，不依赖手输隐藏 URL |
| 真实操作 | 页面按钮或表单能触发 server action / API，产生数据库状态变化 |
| 权限 | 未授权角色无法通过菜单、直达 URL、AI 工具或 server action 越权 |
| 审计 | 关键写操作能在操作日志中追溯操作者、对象、结果摘要 |
| 异常处理 | 校验失败、权限拒绝、第三方失败和空数据有明确提示 |
| 验证证据 | 记录命令输出、截图、日志 ID、测试账号或数据库证据位置 |

## 3. P0 必查项

| 检查项 | 验收方法 | 结果 | 证据位置 |
|--------|----------|------|----------|
| 商城客户下单写入审计 | 消费者提交订单后，在 `/dashboard/logs` 查询“提交商城订单” |  |  |
| 商城取消订单写入审计 | 消费者取消订单后，在 `/dashboard/logs` 查询“取消商城订单” |  |  |
| 商城确认收货写入审计 | 消费者确认收货后，在 `/dashboard/logs` 查询“确认商城收货” |  |  |
| 地址和个人资料变更写入审计 | 新增/删除地址、修改昵称后，在 `/dashboard/logs` 查询商城日志 |  |  |
| 管理员 AI 等价工具边界 | 执行 `npm run test:ai-tools`，确认 admin 使用带对象参数的后台等价工具，不直接混用消费者/经销商身份工具 |  |  |
| 非管理员 AI 工具隔离 | 用销售、仓管、财务、消费者、经销商账号抽查越权问题 |  |  |
| 权限策略口径明确 | `/dashboard/settings#permission-policy` 显示固定角色矩阵和不可动态编辑说明 |  |  |
| 配置中心处理闭环 | `/dashboard/settings` 能区分程序完整度、运营验收和外部硬配置 |  |  |
| 报表导出写入审计 | 后台导出订单 CSV、票据 CSV、财务对账单 HTML 后，在 `/dashboard/logs` 查询对应“导出”日志 |  |  |
| 地图降级边界明确 | 未配置或加载高德失败时，`/dashboard/map` 显示本地降级模式且不使用冷灰大块状态 |  |  |

## 4. 当前默认结论规则

| 条件 | 结论 |
|------|------|
| 任一 P0 必查项失败 | 系统完整度不通过 |
| 自动门禁失败 | 系统完整度不通过 |
| 自动门禁通过但模块验收无证据 | 不能判定通过 |
| 自动门禁通过且模块验收有证据 | 可判定系统本体完整 |
