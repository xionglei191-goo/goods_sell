## 2. 测试目标

本方案用于验证华启酒饮数字渠道平台在正式运营前是否具备可用、可控、可追溯的上线条件。

测试目标包括：

1. 验证客户从进入商城、提交需求、下单、支付、履约到售后的主流程可闭环。
2. 验证公司内部商品、库存、采购、订单、配送、财务和对账流程可正常使用。
3. 验证经销商分单、接单、拒单、库存上报和结算流程符合渠道保护策略。
4. 验证线索、询价、报价、推广码、新品推送和 AI 互动能支持运营获客。
5. 验证权限、日志、合规提示、异常处理和数据口径满足试运营要求。
6. 形成上线准入结论，明确必须修复项、可延期项和试运营风险。

## 3. 测试原则

- 先测主链路，再测边界和异常。
- 先测 P0 运营阻断问题，再测 P1/P2 体验与运营问题。
- 所有涉及金额、库存、分单、收款和客户资料的流程必须有可追溯记录。
- 测试数据应与真实业务接近，但不得混入真实客户隐私数据。
- 测试通过不代表直接全面运营，建议先进入封闭试运营。
- 写数据测试必须使用独立测试库，禁止在生产库或当前业务库执行 seed、Phase 5、下单、支付模拟、库存、财务和推广码相关用例。

### 6.1 独立测试库准备与禁止事项

上线前全功能测试必须使用独立测试库。建议流程：

1. 新建 PostgreSQL 测试库，例如 `goods_sell_test`。
2. 复制 `.env.local` 为测试环境配置，并确认 `DATABASE_URL` 指向测试库。
3. 仅对测试库执行 Prisma 迁移和 seed。
4. 验证 seed 后至少包含 4 个内部账号、33 个 SKU、经销商、订单、收款、日志和系统配置数据。
5. 启动测试服务时使用测试环境变量。
6. 使用测试库运行 Phase 5、商城下单、库存、收款、对账、推广码、经销商接单等写数据测试。

如果本机账号没有 `CREATE DATABASE` 权限，可以临时使用独立 schema 执行开发期 smoke：

```powershell
# 在原 DATABASE_URL 基础上增加 ?schema=goods_sell_test
npx.cmd prisma migrate reset --force
```

独立 schema 只适合开发期隔离验证。正式封闭试运营前，仍应由数据库管理员创建独立测试库。

禁止事项：

- 禁止在生产库执行 `prisma db seed`。
- 禁止在当前业务库执行 Phase 5、支付模拟、下单、库存、财务、推广码等写数据测试。
- 禁止使用真实客户手机号、地址、企业税号或个人隐私数据。
- 禁止把测试库账号密码写入公开仓库。

### 6.2 Windows PowerShell 命令注意事项

在 Windows PowerShell 中，`npm.ps1` 和 `npx.ps1` 可能被执行策略拦截。测试命令统一使用：

```powershell
npm.cmd run lint
npx.cmd tsc --noEmit
```

执行独立 `tsx` 数据库脚本前，需要显式加载环境变量。示例：

```powershell
Get-Content .env.local | ForEach-Object {
  if ($_ -match '^\s*([^#][A-Za-z_][A-Za-z0-9_]*)\s*=\s*(.*)$') {
    $v = $matches[2].Trim()
    if (($v.StartsWith('"') -and $v.EndsWith('"')) -or ($v.StartsWith("'") -and $v.EndsWith("'"))) {
      $v = $v.Substring(1, $v.Length - 2)
    }
    [Environment]::SetEnvironmentVariable($matches[1], $v, 'Process')
  }
}
npm.cmd run test:phase6
```

## 21. 上线风险清单

| 风险 | 影响 | 建议 |
