# 华启商城 — 部署方案

## 1. 开发环境

### 1.1 本地环境要求
| 工具 | 版本 | 说明 |
|------|------|------|
| Node.js | 20.x LTS | 运行时 |
| npm/pnpm | latest | 包管理 |
| PostgreSQL | 16.x | 数据库 |
| Git | latest | 版本控制 |

### 1.2 环境变量配置 (.env)
```bash
# 数据库
DATABASE_URL="postgresql://user:password@localhost:5432/huaqi_mall"

# 认证
NEXTAUTH_SECRET="your-secret-key"
NEXTAUTH_URL="http://localhost:3000"
AUTH_SECRET="your-secret-key"

# 高德地图
AMAP_KEY="your-amap-key"
AMAP_SECRET="your-amap-security-js-code"
NEXT_PUBLIC_AMAP_KEY="your-amap-key"
NEXT_PUBLIC_AMAP_SECURITY_CODE="your-amap-security-js-code"

# AI（统一大模型网关，Anthropic /v1/messages 兼容格式）
AI_BASE_URL="https://your-ai-host/v1"
AI_API_KEY="your-ai-key"
AI_MODEL="qwen3.6-plus"
AI_TIMEOUT_MS="60000"

# AI 备用供应商 Key；仅在直连 DashScope 或 DeepSeek 时填写，默认留空
DASHSCOPE_API_KEY=""
DEEPSEEK_API_KEY=""

# 微信支付
WECHAT_PAY_MCH_ID="your-mch-id"
WECHAT_PAY_SERIAL_NO="your-merchant-cert-serial-no"
WECHAT_PAY_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\\n...\\n-----END PRIVATE KEY-----"
WECHAT_PAY_APIV3_KEY="your-32-byte-apiv3-key"
WECHAT_PAY_PLATFORM_CERT="-----BEGIN CERTIFICATE-----\\n...\\n-----END CERTIFICATE-----"
WECHAT_PAY_NOTIFY_URL="https://your-domain.com/api/wechat/pay/notify"

# 票税与合规
TAX_PROVIDER="MOCK"
TAX_API_BASE_URL=""
TAX_API_KEY=""
ALCOHOL_BUSINESS_LICENSE_NO=""

# 应用
NEXT_PUBLIC_APP_NAME="华启商城"
NEXT_PUBLIC_APP_URL="https://your-domain.com"
```

AI 代办助手部署注意事项：

- `/api/ai/assistant` 和 `/api/ai/assistant/confirm` 依赖登录态、`AUTH_SECRET`、AI 网关配置和数据库连接。
- 生产环境必须使用稳定、足够随机的 `AUTH_SECRET`/`NEXTAUTH_SECRET`，确认 token 与登录会话均依赖该密钥。
- AI 网关不可用时，气泡仍会做权限校验，但模型规划和自然语言能力会降级；上线前必须通过 `system_launch_readiness` 或 `npm run check:launch` 检查。
- 票税和酒类资质变量暂可使用 Mock/空值完成内部测试，正式运营前必须按实际服务商和资质补齐。

## 2. 生产部署

### 2.1 VPS 配置建议
| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2核 | 4核 |

| 硬盘 | 40GB SSD | 80GB SSD |
| 带宽 | 5Mbps | 10Mbps |

### 2.2 部署架构
```
Internet → Nginx (443/80)
              ├── /       → Next.js :3000 (PM2)
              ├── /api/   → Next.js :3000
              └── static  → CDN / 本地文件
           PostgreSQL :5432 (本地)
```

### 2.3.1 日常更新脚本
项目内置 VPS 更新脚本，默认部署到当前服务器 `root@103.229.126.92:/data/goods_sell`，默认 PM2 应用名为 `goods_sell`。生产域名为 `shop.xionglei.online`，DNS 由 Cloudflare 托管。当前服务器的 `80` 端口属于另一个网站，华启商城由 PM2 监听 `3005`，日常更新脚本只更新应用目录并重启 PM2，不修改 Nginx 或 `80` 端口配置。

本机可创建 `.env.deploy.local` 保存部署参数；该文件已被 `.gitignore` 忽略，不应提交到仓库。生产建议使用 `DEPLOY_SSH_KEY` 指向本机部署密钥，服务器密码仅允许作为临时 fallback 保存在本机私有文件或密码管理器中。

```bash
npm run deploy:vps
```

如服务器项目目录或 PM2 应用名不同，可通过环境变量覆盖：

```bash
DEPLOY_PATH=/www/wwwroot/goods_sell PM2_APP=goods_sell npm run deploy:vps
```

脚本默认使用发布包上传模式，适合当前服务器这种非 Git 工作副本的目录；如目标目录本身是 Git 仓库，可设置 `DEPLOY_MODE=git`。脚本不会保存服务器密码。建议配置 SSH key；如临时使用密码，可安装 `sshpass` 后通过 `DEPLOY_PASSWORD` 环境变量传入。

### 2.4 Nginx 配置
```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name your-domain.com;
    
    ssl_certificate /etc/ssl/certs/your-cert.pem;
    ssl_certificate_key /etc/ssl/private/your-key.pem;
    
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    location /_next/static {
        alias /var/www/huaqi-mall/.next/static;
        expires 365d;
        access_log off;
    }
}
```

## 4. 监控

- PM2 自带进程监控和日志
- Nginx access/error log
- PostgreSQL 慢查询日志
- AI assistant 调用失败、tool 越权、确认失败和 HIGH_RISK 执行记录应纳入操作日志巡检
- 上线前检查顺序必须先内后外：先执行 `npm run check:system` 作为程序自身完整度自动门禁，再按 `docs/15-系统完整度验收表.md` 留存真实角色、真实操作、权限、审计和异常证据，然后执行 `npm run check:operational` 检查运营接手与人工签收，最后执行 `npm run check:launch` 检查正式硬配置。
- `npm run check:system` 只覆盖系统本体完整度自动门禁：后台、商城、经销商端、AI、权限、订单、库存、财务、营销、微信生态、日志、配置中心、报表、地图配送等模块，并逐项检查真实入口、真实操作、权限、审计、异常处理和验证证据。它不能替代 `docs/15-系统完整度验收表.md` 的人工证据；域名、微信、小程序、支付、税控、酒类资质不属于程序完整度缺陷。
- `npm run check:operational` 覆盖业务负责人签收、价格复核、库存盘点、账号权限复核、真实设备/微信内置浏览器、真实小额支付、小程序体验版、税控联调和备份恢复演练；正式上线前需按 `docs/14-正式上线验收记录模板.md` 留痕。
- `npm run check:launch` 默认按正式公开上线口径检查数据库、Auth、公网地址、AI、地图、微信、小程序、支付、税控和酒类经营资质；这些是上线外部硬条件，不应作为系统开发未完成的理由。封闭试运营可显式使用 `LAUNCH_READINESS_MODE=trial`。
- 后续可接入 Uptime Robot 等外部监控
