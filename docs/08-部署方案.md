# 华启商城 — 部署方案

## 1. 开发环境

### 1.1 本地环境要求
| 工具 | 版本 | 说明 |
|------|------|------|
| Node.js | 20.x LTS | 运行时 |
| npm/pnpm | latest | 包管理 |
| PostgreSQL | 16.x | 数据库 |
| Git | latest | 版本控制 |

### 1.2 环境变量配置 (.env)
```bash
# 数据库
DATABASE_URL="postgresql://user:password@localhost:5432/huaqi_mall"

# 认证
NEXTAUTH_SECRET="your-secret-key"
NEXTAUTH_URL="http://localhost:3000"

# 高德地图
AMAP_KEY="your-amap-key"
AMAP_SECRET="your-amap-security-js-code"

# AI（统一大模型网关，Anthropic /v1/messages 兼容格式）
AI_BASE_URL="https://your-ai-host/v1"
AI_API_KEY="your-ai-key"
AI_MODEL="qwen3.6-plus"
AI_TIMEOUT_MS="60000"

# AI 备用供应商 Key；仅在直连 DashScope 或 DeepSeek 时填写，默认留空
DASHSCOPE_API_KEY=""
DEEPSEEK_API_KEY=""

# 微信支付
WECHAT_PAY_MCH_ID="your-mch-id"
WECHAT_PAY_API_KEY="your-api-key"
WECHAT_PAY_CERT_PATH="/path/to/cert"

# 应用
NEXT_PUBLIC_APP_NAME="华启商城"
NEXT_PUBLIC_APP_URL="https://your-domain.com"
```

## 2. 生产部署

### 2.1 VPS 配置建议
| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2核 | 4核 |
| 内存 | 4GB | 8GB |
| 硬盘 | 40GB SSD | 80GB SSD |
| 带宽 | 5Mbps | 10Mbps |
| 系统 | Ubuntu 22.04 | Ubuntu 22.04 |

### 2.2 部署架构
```
Internet → Nginx (443/80)
              ├── /       → Next.js :3000 (PM2)
              ├── /api/   → Next.js :3000
              └── static  → CDN / 本地文件
           PostgreSQL :5432 (本地)
```

### 2.3 部署步骤
```bash
# 1. 安装依赖
sudo apt update
sudo apt install -y nginx postgresql nodejs npm

# 2. 配置 PostgreSQL
sudo -u postgres createuser huaqi
sudo -u postgres createdb huaqi_mall -O huaqi

# 3. 克隆项目
git clone <repo-url> /var/www/huaqi-mall
cd /var/www/huaqi-mall

# 4. 安装依赖
npm install

# 5. 环境变量
cp .env.example .env
# 编辑 .env 填入生产环境配置

# 6. 数据库迁移
npx prisma migrate deploy
npx prisma db seed

# 7. 构建
npm run build

# 8. PM2 启动
npm install -g pm2
pm2 start npm --name "huaqi-mall" -- start
pm2 save
pm2 startup

# 9. Nginx 配置
# 见下方配置文件
```

### 2.3.1 日常更新脚本
项目内置 VPS 更新脚本，默认部署到 `root@103.229.126.92:/var/www/huaqi-mall`，默认 PM2 应用名为 `huaqi-mall`。

```bash
npm run deploy:vps
```

如服务器项目目录或 PM2 应用名不同，可通过环境变量覆盖：

```bash
DEPLOY_PATH=/www/wwwroot/goods_sell PM2_APP=goods-sell npm run deploy:vps
```

脚本不会保存服务器密码。建议配置 SSH key；如临时使用密码，可安装 `sshpass` 后通过 `DEPLOY_PASSWORD` 环境变量传入。

### 2.4 Nginx 配置
```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name your-domain.com;
    
    ssl_certificate /etc/ssl/certs/your-cert.pem;
    ssl_certificate_key /etc/ssl/private/your-key.pem;
    
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    location /_next/static {
        alias /var/www/huaqi-mall/.next/static;
        expires 365d;
        access_log off;
    }
}
```

## 3. 备份策略

```bash
# PostgreSQL 每日备份
0 2 * * * pg_dump huaqi_mall | gzip > /backup/huaqi_mall_$(date +\%Y\%m\%d).sql.gz

# 保留最近30天
find /backup -name "huaqi_mall_*.sql.gz" -mtime +30 -delete
```
