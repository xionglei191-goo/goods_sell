# 华启酒饮数字渠道平台 — 项目概述

## 1. 项目背景

华启公司是湘潭市区域内多品牌酒类、食品、饮料的总代理/总经销商。公司通过销售团队将产品批发给本市各零售商和经销商，已建立起成熟的线下分销网络。

当前面临的机遇与挑战：
- **线下渠道趋于饱和**：本市约500家经销商已基本覆盖
- **终端需求未沉淀**：终端客户、宴席、企业团购、门店补货需求大量停留在业务员或经销商私域
- **获客成本上升**：传统地推方式效率递减
- **渠道保护压力**：直接做线上零售容易让下游经销商认为平台抢单
- **管理效率待提升**：进销存、财务对账仍依赖手工或简单工具
- **操作门槛较高**：老板、老年客户和部分经销商不熟悉复杂菜单，需要能用自然语言完成查询、下单和管理代办

## 2. 项目目标

### 2.1 短期目标（3个月）
- 将平台定位从“线上商城”修正为“区域总代数字化获客与渠道协同系统”
- 建立业务员/经销商专属推广入口，沉淀扫码、询价、互动和下单线索
- 跑通“小单经销商承接、大单公司/业务员报价”的渠道保护闭环
- 实现进销存数字化管理
- 建立基础的客户管理和财务对账能力

### 2.2 中期目标（6个月）
- 通过业务员地推、经销商私域、小程序/公众号和本地内容渠道获客，月活用户达到 5000+
- 上线智能分单系统，打通经销商履约网络
- 引入 AI 选品互动、询价助手、用户画像系统和权限化 AI tools 代办入口
- 上线微信小程序并完成真实 AppID、支付、公众号模板消息配置

### 2.3 长期目标（12个月）
- 覆盖湘潭全市消费者，月活达到 20000+
- 完善 AI 画像和新品推送体系，实现按场景、区域、门店类型精准触达
- 经销商系统全面上线，形成数字化供应链
- 探索 AI 直播等创新营销模式

## 3. 项目范围

### 3.1 在范围内
| 模块 | 说明 |
|------|------|
| 场景化客户入口 | 面向宴席配酒、企业团购、门店补货、普通散客的下单/询价入口 |
| 线索与询价 | AI互动、地推扫码、经销商推荐先沉淀线索，再按小单/大单分流 |
| 进销存管理 | 产品、库存、采购管理 |
| 销售管理 | 订单、客户、销售员业绩管理 |
| 财务对账 | 应收/应付、收款核销、对账单 |
| 渠道保护分单 | 基于地图、订单金额、商品阈值、经销商政策的订单路由分配 |
| AI 选品互动 | 宴席配酒、送礼推荐、门店补货、趣味测评等需求采集入口 |
| AI 代办助手 | 全端 AI 悬浮气泡，按角色权限调用 tools 完成查询、草稿、确认和执行 |
| 用户画像 | 客户标签、行为分析、场景偏好、预算和品牌偏好 |
| 业务员/经销商协同 | 推广码、门店链接、接单/拒单、库存上报、结算 |
| 管理后台 | 多角色的后台管理系统 |

### 3.2 不在范围内（后续迭代）
- 原生移动 APP（iOS/Android）
- 对外售卖的通用 SaaS 平台
- AI 直播系统
- ERP 深度集成
- 多仓库管理

## 4. 目标用户

| 用户群体 | 画像 | 使用场景 |
|---------|------|---------|
| 宴席/团购客户 | 婚宴、寿宴、升学宴、企业福利采购负责人 | 询价、预算测算、套餐推荐、开票配送咨询 |
| 门店补货客户 | 烟酒店、小超市、餐饮店老板 | 快速补货、看新品、询批发价、赊账/对账 |
| 普通散客 | 湘潭市居民，25-65岁，中老年为主 | 场景化选品、下单、优惠券、AI互动 |
| 经销商(B端) | 本市零售店/便利店/超市老板 | 承接附近小单、推荐客户扫码、在线进货、对账 |
| 管理员 | 公司管理层，包含不熟悉复杂后台菜单的老板 | 全局管控、数据分析、系统配置、通过 AI 代办发起管理员操作 |
| 销售员 | 公司销售团队 | 地推经销商、推广码、客户跟进、询价报价、业绩查看 |
| 仓库管理员 | 仓库工作人员 | 入库/出库/盘点操作 |
| 财务 | 公司财务人员 | 应收应付管理、对账、报表 |

## 5. 商业价值

1. **开辟新入口**：从纯线下批发扩展到线上线索、询价和订货入口
2. **保护渠道利益**：小单回流经销商，大单公司统筹，降低渠道抵触
3. **提高运营效率**：数字化管理替代手工操作
4. **数据资产积累**：用户画像和消费数据是长期核心资产
5. **供应链优化**：智能分单降低物流成本，提升履约效率
6. **品牌建设**：从传统批发商向区域品牌商转型
7. **新品推广提效**：按客户画像、门店类型和区域选择推新品对象
8. **降低使用门槛**：把“人找菜单操作”升级为“自然语言发起任务”，减少客户、经销商和老板重复录入

## 6. 风险与应对

| 风险 | 级别 | 应对策略 |
|------|------|---------|
| C端获客困难 | 高 | 业务员地推+经销商私域二维码+AI互动引流+本地内容渠道 |
| 经销商抵触系统 | 高 | 明确小单回流经销商，增加经销商政策、价格权限、投诉台账 |
| 技术复杂度高 | 中 | 分阶段实施，核心功能优先 |
| 酒类线上销售资质 | 高 | 提前办理食品经营许可证及酒类流通许可 |
| 支付安全 | 中 | 使用微信支付官方接口，严格遵守规范 |
| 个人信息合规 | 高 | 在收集手机号、地址、偏好前明示用途、范围、保存期限和用户权利 |
| 酒类广告合规 | 高 | 避免诱导饮酒、夸大功效、面向未成年人营销，增加禁售和适度饮酒提示 |
| AI 误操作或越权 | 高 | AI tools 复用服务端权限守卫；写操作确认后执行，高风险操作二次确认并写审计日志 |
