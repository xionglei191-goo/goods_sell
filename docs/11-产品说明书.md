## 2. 产品概述

华启酒饮数字渠道平台面向湘潭区域酒水、食品、饮料总代业务，帮助公司把线下业务员地推、经销商私域、终端客户询价、门店补货、宴席团购和新品推广沉淀到统一系统。

平台不是单纯的 B2C 商城，也不是对外售卖的通用 SaaS。它的核心价值是：

1. 将客户需求从业务员和经销商私域中沉淀为可跟进的线索、询价和订单。
2. 通过“小单就近经销商承接、大单公司或业务员报价”的方式保护原有渠道。
3. 用进销存、采购、配送、财务和对账能力提升公司内部运营效率。
4. 用 AI 互动、客户画像、新品推送和推广码追踪提升获客与复购效率。
5. 用经销商政策、分单记录和渠道冲突台账降低跨区、低价、拒单等管理风险。

## 6. 核心业务流程

### 6.1 客户线索沉淀流程

1. 客户通过业务员二维码、经销商门店码、H5商城、小程序、公众号或 AI 互动进入平台。
2. 客户选择场景：宴席配酒、企业团购、门店补货、普通购买、经销商入驻或新品试饮。
3. 平台采集姓名、手机号、预算、地址、偏好、配送时间、开票需求等必要信息。
4. 系统生成线索，记录来源、场景、推广码、关联销售员或经销商。
5. 销售员或经销商跟进线索，更新状态为已分派、跟进中、已转化或已流失。
6. 高意向线索可继续转为询价单、报价单或正式订单。

### 6.2 询价与报价流程

1. 客户在场景页或 AI 互动中提交需求。
2. 系统生成询价单，保留预算、日期、地址、商品意向、开票需求和备注。
3. 管理员或销售员分派负责人，也可关联经销商处理。
4. 负责人创建报价单，配置商品组合、总金额、有效期和报价说明。
5. 报价状态支持草稿、已发送、已接受、已拒绝、已过期、已转订单。
6. 客户确认后，报价单可转为订单并进入履约和收款流程。

### 6.3 普通下单流程

1. 客户进入 `/shop` 浏览分类、热门推荐、新品和商品详情。
2. 客户加入购物车并选择收货地址。
3. 系统计算订单金额、优惠、应付金额和配送信息。
4. 客户提交订单，选择微信、现金、转账或赊账等支付方式。
5. 系统根据订单金额、商品阈值、地址和经销商政策确定公司仓库履约或经销商履约。
6. 订单进入待支付、已支付、已确认、配送中、已完成、取消或退款状态。

### 6.4 渠道保护分单流程

1. 商品可配置散单/大单阈值，订单金额和订单类型也参与判断。
2. 小额零售订单优先匹配服务半径内、正在接单、有库存能力、政策允许的经销商。
3. 大额订单、团购订单和复杂询价优先进入公司或业务员报价流程。
4. 经销商可接单或拒单，拒单后进入下一家匹配或转公司处理。
5. 分单过程记录距离、原因、状态和响应时间。
6. 跨区、低价、频繁拒单、库存不符或客户投诉进入渠道冲突台账。

### 6.5 进销存流程

1. 商品基础资料由后台维护，包括 SKU、条码、分类、品牌、规格、单位和价格。
2. 采购单由采购或仓管创建，关联供应商和采购明细。
3. 到货后执行入库，系统更新商品库存并生成库存流水。
4. 销售发货或手工调整时执行出库或调整。
5. 定期盘点生成盘点单，录入实盘数量后确认差异。
6. 低于安全库存的商品进入预警，辅助采购和补货。

### 6.6 财务对账流程

1. 订单生成应收或收款记录。
2. 财务按客户、订单或线下收款登记回款。
3. 系统支持现金、转账、微信、赊账等支付方式。
4. 财务核销订单，更新欠款、已收和未收状态。
5. 按客户和时间段生成对账单。
6. 如需开票，登记发票类型、抬头、税号、金额和关联订单/收款。

## 7. 功能模块说明

### 7.1 管理后台首页

后台首页为公司经营看板，展示订单、销售额、新增客户、待处理事项、销售趋势、订单状态、库存预警、应收账款、销售员排行、经销商经营、客户类型、新品推送和渠道冲突趋势。

适用场景：
- 管理层查看日常经营情况。
- 运营人员发现待处理订单、库存预警和渠道异常。
- 销售负责人查看业务员绩效和线索转化。

### 7.2 商品与图片素材管理

商品管理支持：
- 商品列表、搜索、筛选、排序。
- 新增、编辑、查看商品详情。
- SKU、条码、名称、分类、品牌、单位、规格。
- 成本价、批发价、零售价、会员价。
- 当前库存、安全库存、散单/大单阈值。
- 商品上下架和缺货状态。
- 商品图片和图片素材采集、审核、授权状态记录。
- 分类树和品牌资料管理。

产品分类建议按酒类、食品、饮料三级结构维护，以便商城端筛选和后台报表统计。

### 7.3 库存与仓库管理

库存模块支持：
- 库存总览、库存价值和低库存预警。
- 入库：采购入库、退货入库、库存调整。
- 出库：订单出库、手工出库、损耗出库。
- 盘点：创建盘点单、录入实盘、确认差异。
- 库存流水：按商品、类型、操作人和时间查询。

仓库模块用于跟踪库存盘点、仓库作业和库存差异，适合仓管日常使用。

### 7.4 采购与供应商管理

采购模块支持：
- 供应商资料维护，包括名称、联系人、电话、地址和启用状态。
- 采购订单创建、提交、到货、完成、取消。
- 采购明细维护，包括商品、数量、已收数量、采购单价和金额。
- 采购入库后联动库存变化。

### 7.5 客户管理

客户管理支持：
- 客户列表、客户类型筛选、销售员筛选、标签筛选和动态分层。
- 客户详情，包括档案、订单历史、欠款、画像、标签和拜访记录。
- 客户标签，如宴席客户、企业团购、餐饮采购、烟酒店补货、高价值客户、沉默客户等。
- 销售员客户归属和后续跟进记录。

客户画像由订单、互动、询价、偏好、频次和标签共同构成，用于推荐、新品推送和销售跟进。

### 7.6 经销商管理

经销商模块支持：
- 经销商列表、动态分层、接单指标、库存指标和推广指标。
- 经销商详情，包括门店信息、位置、接单记录、结算、绑定业务员。
- 经销商政策配置：服务半径、最小/最大接单金额、价格等级、是否允许跨区、是否允许拒单、每日拒单上限、优先级和可接品牌。
- 地图总览：经销商标注、订单热力图和区域覆盖情况。

经销商既是渠道伙伴，也是系统中的特殊客户。经销商通过客户账号关联经销商资料，便于统一管理订单、结算和客户关系。

### 7.7 订单与配送管理

订单模块支持：
- 订单列表、多维筛选和导出。
- 订单详情：客户、地址、商品明细、金额、支付、配送、分单记录和备注。
- 手动开单：适用于线下电话订货、业务员代客开单和内部补录。
- 状态维护：待支付、已支付、已确认、配送中、已完成、已取消、退款中、已退款。
- 分单信息：经销商、距离、分派原因、接单状态和响应时间。

配送模块支持：
- 配送单列表。
- 发货、配送中、已送达状态维护。
- 快递单号或配送方式记录。

### 7.8 线索、询价与报价

渠道经营模块支持：
- 线索池：管理来自商城、微信、业务员码、经销商码、AI 互动和手工录入的线索。
- 询价单：承接宴席、团购、门店补货、礼品、新品试饮等需求。
- 报价单：为询价生成商品组合、报价金额、有效期和成交记录。
- 报价转订单：客户确认后形成正式订单。
- 推广码：为业务员、经销商或活动生成专属码，统计扫码、线索和订单。
- 地推试点：批量绑定业务员和经销商，生成地推码和门店码。

### 7.9 商城与场景化订货

商城端支持：
- 首页：区域供货定位、渠道保护说明、场景入口、推荐商品、热门商品、新品上架。
- 商品目录：分类浏览、搜索、筛选、排序。
- 商品详情：图片、规格、价格、库存、购买操作。
- 购物车：商品数量、选中、合计。
- 结算：地址选择、支付方式、备注和提交订单。
- 我的订单：订单列表、详情、取消、退款等操作入口。
- 账户中心：个人资料、地址、优惠券和积分。

场景入口包括：
- 宴席配酒：采集桌数、预算、品牌偏好、饮料搭配、配送时间。
- 企业团购：采集预算、收礼对象、包装、开票和配送批次。
- 门店补货：采集门店类型、补货品类、周转速度和价格敏感度。
- 经销商/业务员入口：识别推广来源，记录绑定关系。
- AI 选品互动：通过对话采集需求并生成建议。
- 趣味互动：节气养生、AI 测试、签到和新品试饮资格。

### 7.10 经销商端

经销商端支持：
- 待接订单：查看系统分派的附近小单。
- 我的订单：查看已接订单和处理进度。
- 库存上报：按商品上报门店库存，供分单时参考。
- 结算中心：查看佣金、结算明细和历史记录。
- 门店推广：生成门店专属二维码/链接，查看扫码客户和成交统计。
- 门店线索：查看来自门店码、分派到本店或与本店相关的客户线索。

### 7.11 财务、收款与票据

财务模块支持：
- 财务总览：收入、应收、趋势和关键指标。
- 应收账款：按客户查看欠款、账龄和订单来源。
- 收款登记：登记回款并核销订单。
- 对账单：按客户和时间段生成对账结果。
- 票据：记录普通发票、专用发票、开票状态和关联金额。

### 7.12 营销与新品推送

营销模块支持：
- 活动管理：草稿、进行中、暂停、完成。
- 优惠券：满减券、折扣券、发放数量、使用数量、有效期。
- 新品推送：按客户画像、客户标签或活动目标选择客户，生成话术并跟踪发送、打开、点击和转化。
- 运营复盘：查看不同客户层级、经销商区域和商品组合的转化效果。

### 7.13 AI 互动与客户画像

AI 模块支持：
- 宴席配酒、企业团购、送礼推荐、门店补货、新品试饮和知识问答。
- 对话内容沉淀为线索、询价建议、客户标签和偏好。
- 用户画像分析：消费等级、偏好分类、购买频率、生命周期和标签。
- AI 内容缓存，降低重复生成成本并提升响应速度。

AI 输出用于辅助销售与运营决策，不应替代最终报价、合同和合规审核。

### 7.14 微信生态

微信能力包括：
- 小程序登录、首页、目录、购物车、地址、下单、订单和分享接口。
- 公众号菜单配置和消息记录。
- 微信支付预下单与支付通知接口。

上线前需要配置真实 AppID、AppSecret、商户号、证书、回调地址、服务器域名和微信平台类目资质。

### 7.15 系统设置与操作日志

系统设置支持：
- 业务参数配置。
- 用户账号管理。
- 系统开关和分组配置。

操作日志记录：
- 操作人、模块、动作、对象、前后变化、摘要、IP、UserAgent 和时间。
- 用于追责、审计、问题排查和关键业务回溯。

## 9. 状态与口径

### 9.1 订单状态

| 状态 | 含义 |
|------|------|
| 待支付 | 订单已创建，客户尚未付款或待确认 |
| 已支付 | 客户完成在线支付或线下支付已登记 |
| 已确认 | 公司、仓库或经销商确认订单可履约 |
| 配送中 | 商品已出库或正在配送 |
| 已送达 | 商品已送达客户 |
| 已完成 | 订单流程结束 |
| 已取消 | 订单取消，不再履约 |
| 退款中 | 正在处理退款 |
| 已退款 | 退款完成 |

### 9.2 线索状态

| 状态 | 含义 |
|------|------|
| 新线索 | 刚进入系统，尚未跟进 |
| 已分派 | 已分给销售员或经销商 |
| 跟进中 | 已开始沟通 |
| 已转化 | 已转询价、报价、订单或成交客户 |
| 已流失 | 无效、拒绝或长期无法联系 |

### 9.3 询价和报价状态

询价状态包括新建、已分派、已报价、已成交、已流失和已取消。报价状态包括草稿、已发送、已接受、已拒绝、已过期和已转订单。

### 9.4 分单口径

- 经销商履约：订单分派给经销商，由经销商承接配送或服务。
- 仓库履约：订单由公司仓库或公司业务团队统一处理。
- 大单：按商品散单/大单阈值、订单金额、团购类型和人工策略判断。
- 小单：优先使用附近经销商履约，以保护渠道利益。

### 9.5 财务口径

- 应收金额：订单应收或客户欠款。
- 已收金额：已完成支付或已登记回款。
- 未收金额：应收金额减去已收金额。
- 收款方式：微信、现金、转账、赊账。
- 对账单：按客户、时间段和订单/收款记录汇总。

## 11. 典型使用场景

### 11.1 宴席客户询价

1. 客户扫码进入宴席配酒页。
2. 填写宴席类型、桌数、预算、日期、地址和偏好。
3. 系统生成线索和询价单。
4. 业务员创建报价单。
5. 客户确认后报价转订单。
6. 仓库配送或公司安排履约。
7. 财务收款、开票和对账。

### 11.2 经销商承接附近小单

1. 客户在商城提交普通小单。
2. 系统根据地址、金额、商品、库存和政策匹配经销商。
3. 经销商端收到待接订单。
4. 经销商确认接单并配送。
5. 系统记录分单、履约和结算。
6. 如经销商拒单，系统转派下一家或回到公司处理。

### 11.3 业务员地推沉淀客户

1. 管理员为业务员生成推广码。
2. 业务员拜访经销商或终端客户时引导扫码。
3. 客户提交需求或浏览商品。
4. 系统记录推广来源并生成线索。
5. 业务员在后台跟进、报价和转订单。
6. 管理层在业务员绩效中查看扫码、线索、报价和成交数据。

### 11.4 门店补货

1. 门店老板进入门店补货页或 AI 补货助手。
2. 选择门店类型、畅销品类、补货预算和配送需求。
3. 系统生成补货建议和询价。
4. 经销商或销售员确认价格与库存。
5. 形成订单并完成配送。
6. 后续根据补货频率和品类偏好进行新品推送。

### 11.5 新品精准推送

1. 运营选择新品或活动。
2. 按客户标签、消费等级、门店类型或区域筛选目标客户。
3. 系统生成推送话术。
4. 记录发送、打开、点击、咨询、试饮和成交。
5. 运营根据转化数据复盘，并优化下一轮推送对象。

## 13. 合规与风控说明

### 13.1 酒类经营与广告

- 平台上线前应确认食品经营许可、酒类流通相关资质和本地监管要求。
- 页面应保留未成年人禁售、理性饮酒和适度饮酒提示。
- 营销内容不得诱导过量饮酒，不得暗示饮酒带来健康、社交或能力提升。
- AI 推荐不得生成夸大功效或面向未成年人的酒类营销内容。

### 13.2 个人信息保护

- 收集手机号、地址、偏好、画像标签前应明示用途、范围和保存方式。
- 客户应可查看和修改自己的基础资料、地址和授权信息。
- 推广码追踪应仅用于业务归因、客户服务和合法营销。
- 导出数据、客户资料和对账单应限制权限并保留操作日志。

### 13.3 支付与财务

- 微信支付上线需使用官方商户号、证书、签名和回调验签。
- 线下现金、转账和赊账需由财务登记并核销。
- 开票信息和对账单涉及敏感经营数据，应限制访问。

### 13.4 渠道风险

- 平台应明确小单回流经销商，大单由公司统筹，避免渠道误解。
- 对跨区、低价、拒单、库存不符和投诉建立台账。
- 经销商政策调整应有记录，避免分单争议。

## 16. 运维与数据维护

### 16.1 日常维护

- 每日检查待处理订单、库存预警、待跟进线索和渠道冲突。
- 每周复盘业务员推广码、经销商承接率和询价转化。
- 每月核对库存、应收账款、经销商结算和供应商往来。
- 定期清理无效测试账号、重复客户、无效商品图片和过期优惠券。

### 16.2 数据备份

- 生产数据库应配置每日自动备份。
- 重要版本上线前应执行手动备份。
- 商品、客户、订单、库存、收款和日志数据需要长期保存。
- 图片素材应保留来源、授权状态和本地/云存储地址。

### 16.3 异常处理

| 异常 | 处理方式 |
|------|----------|
| 支付成功但订单未更新 | 检查微信回调、交易号和支付日志，必要时手工核销 |
| 经销商未响应 | 超时转派下一家或回公司处理 |
| 库存不一致 | 发起盘点，确认差异后调整库存 |
| 客户重复 | 合并客户前核对手机号、订单和收款 |
| 低价或跨区投诉 | 进入渠道冲突台账，分派责任人处理 |
| AI 推荐异常 | 人工复核，不直接用于报价或承诺 |

## 17. 验收标准

产品交付验收建议按以下标准执行：

1. 管理员、销售员、仓管、财务、经销商和客户可按角色进入对应端。
2. 商品、分类、品牌、库存、供应商和经销商基础资料可正常维护。
3. 商城端可完成浏览、加购、结算、地址维护和订单查看。
4. 宴席、团购、门店补货可生成线索和询价。
5. 报价单可创建、发送并转订单。
6. 小单可按经销商政策生成分单记录，经销商可接单或拒单。
7. 库存入库、出库和盘点可生成正确流水。
8. 财务可登记收款、查看应收和生成对账单。
9. 推广码可统计扫码、线索和订单。
10. 新品推送和优惠券可完成基础运营闭环。
11. 关键操作可在操作日志中追溯。
12. 合规提示、个人信息告知和酒类禁售提示在客户侧可见。

| 仓管 | `/dashboard` | 产品只读、库存、采购、仓储、配送和订单履约 |
| 财务 | `/dashboard` | 订单/客户只读、收款、票据、财务总览和对账单 |
| 经销商 | `/dealer/incoming` | 自己门店的接单、订单、库存、结算、推广和线索 |
| 消费者 | `/shop` | 商城浏览、购物车、结算、订单、账户、优惠券和 AI 互动 |

经销商注册不是即时开通，而是“提交申请 → 管理员审核 → 创建门店档案 → 开通登录”。审核前经销商账号处于待认证状态，不能进入 `/dealer/*`。

无权限访问时，系统展示 `/forbidden`，提示当前角色和可进入入口；未登录访问受保护页面时仍跳转登录页。
