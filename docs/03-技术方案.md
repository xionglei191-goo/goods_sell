### 2.1 AI Tools 执行架构

```
AiFloatingBubble
      │
      ├─ GET /api/ai/quick-prompts：按角色加载固定高频词条
      │
      ▼
POST /api/ai/assistant  ── SSE(delta/card/done/error)
      │
      ▼
assistant-service
      │
      ├─ context：读取当前用户、角色、账号类型和数据范围
      ├─ intent-templates：固定词条本地命中，按角色隔离展示与执行
      ├─ planner：保守本地规则转 tool plan，避免宽泛关键词抢占复杂意图
      ├─ provider：非固定或复杂问法调用真实 AI provider，返回 JSON plan
      ├─ registry：统一声明 tool、schema、权限、风险等级和 handler
      ├─ executor：参数校验、权限校验、确认卡、handler 调用和错误归一
      ├─ audit：AI 调用与写操作审计，敏感字段脱敏
      └─ prompt-usage：记录非固定高频问法候选，人工审核后再沉淀
             │
             ▼
       features/* actions / queries / Prisma
```

执行约束：

- AI 不直接越过业务层写库，所有写操作复用既有 action、query、权限守卫和 `logAction()`。
- 固定词条只覆盖高频、低歧义场景，且按角色返回；消费者、经销商、管理员、销售员、仓管和财务看到的词条不同。
- 自由输入遵循“固定词条 → 高置信本地规则 → 真实 AI provider → tool plan 校验 → executor”的链路，避免本地宽泛关键词抢占真实 AI 判断。
- `/api/ai/assistant/confirm` 使用 HMAC confirmation token 绑定用户、角色、tool、参数、风险等级和 10 分钟有效期，防止前端篡改参数。
- `READ` tool 可直接返回结果；`DRAFT` tool 只生成草稿；`WRITE` 和 `HIGH_RISK` tool 必须经过确认卡。
- `HIGH_RISK` tool 还要求输入指定确认文字，适用于禁用账号、重置密码、系统参数和财务确认等操作。
- 非固定问法会记录到 `AiContentCache` 的 `ai_prompt_usage:*` 逻辑键中，只作为高频候选，不自动启用规则。

## 4. 关键技术决策

### 4.1 全栈 Next.js（非前后端分离）
- 利用 Server Components 减少客户端 JS 体积
- Server Actions 处理数据变更，无需写 REST API
- 统一部署，降低运维复杂度

### 4.2 PostgreSQL + Prisma
- Decimal 类型精确处理金额
- JSONB 类型存储灵活数据（用户画像标签等）
- Prisma 提供类型安全和自动迁移

### 4.3 移动优先响应式
- 第一阶段不做原生APP，用响应式Web覆盖手机端
- 重点优化微信内置浏览器体验
- 后续可通过 Taro/uni-app 转小程序

### 4.4 权限化 AI 代办
- 全端只保留一个 AI 悬浮入口，按当前角色动态过滤可用 tools。
- 悬浮入口下方的固定词条由 `/api/ai/quick-prompts` 按角色返回，不使用统一硬编码列表。
- Tool registry 是 AI 能力边界，新增能力必须先声明 schema、权限、风险等级和确认策略。
- 购买历史查询独立为 `customer_purchase_history`，与下单/后台开单拆分，防止“某客户买了什么”误触发写操作草稿。
- 用户体验上用结果卡和确认卡替代复杂表单，服务端仍以 RBAC 和业务 action 作为最终安全边界。
- AI provider 保留 Anthropic-compatible 接口；模型不支持原生 tool calling 时使用 JSON tool plan 降级。

### 4.5 上线就绪检查
- `src/features/system/launch-readiness.ts` 汇总数据库、认证、AI、地图、微信、票税和酒类资质配置检查。
- `npm run check:launch` 用于部署前命令行验收。
- `system_launch_readiness` tool 允许管理员在 AI 气泡中直接询问“现在上线还差什么配置”。
