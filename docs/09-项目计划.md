# 华启酒饮数字渠道平台 — 项目计划

## 总体里程碑

| 阶段 | 名称 | 周期 | 核心交付 |
|------|------|------|---------|
| Phase 1 | 基础框架 + 进销存 | 第1~4天 | 项目骨架、数据库、后台布局、产品/库存管理 |
| Phase 2 | 客户入口 + 线上订货 | 第5~8天 | 商城前端、购物车、下单支付、用户中心 |
| Phase 3 | 销售管理 + 财务 | 第9~12天 | 订单全流程、智能分单、地图、财务对账 |
| Phase 4 | AI + 营销 | 第13~16天 | AI选品互动、用户画像、趣味互动、精准营销 |
| Phase 5 | 微信生态 + 优化 | 后续 | 小程序、公众号、性能优化、全面测试 |
| Phase 6 | 运营支撑模块 | 已完成 | 仓储、配送、票据、设置、日志、部署验证 |
| Phase 7 | 渠道经营改版 | 30天 | 定位改版、场景入口、线索/询价、推广码、渠道保护规则 |
| Phase 8 | 地推与经销商承接 | 90天 | 10-30个经销商试点、大单报价、新品推送、冲突台账 |
| Phase 9 | 渠道资产沉淀 | 180天 | 经销商分层、客户分层、自动化分单、经营看板 |

## 详细时间线

### Phase 1 (Day 1-4)
- Day 1: 项目初始化 + 数据库设计
- Day 2: 认证系统 + 后台布局
- Day 3: 产品管理 + 分类品牌
- Day 4: 库存管理 + 采购基础

### Phase 2 (Day 5-8)
- Day 5: 商城首页 + 产品目录
- Day 6: 产品详情 + 购物车
- Day 7: 下单流程 + 支付对接
- Day 8: 用户中心 + 订单查看

### Phase 3 (Day 9-12)
- Day 9: 订单管理后台
- Day 10: 智能分单 + 地图集成
- Day 11: 客户管理 + 经销商端
- Day 12: 财务对账 + 报表

### Phase 4 (Day 13-16)
- Day 13: AI选品互动基础
- Day 14: 用户画像 + 标签
- Day 15: 趣味互动模块
- Day 16: 精准营销 + 数据仪表盘

### Phase 7 (30天)
- Week 1: 首页与文案从“商城”改为“区域供货 + 渠道保护 + AI选品”
- Week 1: 增加未成年人禁售、适度饮酒、个人信息告知同意等合规提示
- Week 2: 上线宴席配酒、企业团购、门店补货、经销商/业务员四个入口
- Week 2: 新增 Lead、Inquiry、Quote、PromoterCode、DealerPolicy 数据模型草案与迁移
- Week 3: 小单/中单/大单分流规则升级，补充经销商金额范围、品牌权限和拒单转派
- Week 3: AI互动 MVP：宴席预算、送礼推荐、门店补货三个场景
- Week 4: 后台线索池、询价单、推广码统计、渠道冲突台账 MVP

### Phase 8 (90天)
- 业务员试点绑定 10-30 个经销商，验证专属二维码和小单回流
- 经销商端补齐接单、拒单、库存上报、门店推广统计
- 大单/团购从购物车直付改为询价报价流程
- AI画像标签按宴席、团购、餐饮、烟酒店、普通散客分类
- 选择一款新品做画像推送试验，跟踪打开、试饮、下单、复购

### Phase 9 (180天)
- 经销商分层：活跃、普通、待激活、风险经销商（已用动态指标上线 MVP）
- 客户分层：高价值团购、餐饮店、烟酒店、宴席客户、普通消费者
- 新品推广闭环：画像筛选 → AI话术 → 推送 → 试饮/下单 → 复盘
- 分单策略从距离优先升级为距离 + 库存 + 服务能力 + 历史响应
- 公司经营看板聚合区域、经销商、新品、客户类型和业务员地推效果

## 风险缓冲
- 每个Phase预留20%缓冲时间
- 复杂功能可降级处理（如支付先做模拟）
- 图片素材缺失不阻塞开发（使用占位图）
- 涉及酒类广告、个人信息、微信支付和经营资质的上线项必须先做合规检查
