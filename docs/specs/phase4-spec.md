# Phase 4：AI + 营销 — 开发规格书

> 负责人：代码组 | 审核人：架构师 | 工期：Day 13~16

## Step 4.1 AI 智能客服（Day 13）

### 开发说明
1. **API 接入**：默认使用统一大模型网关（Anthropic `/v1/messages` 兼容格式，`AI_BASE_URL` / `AI_API_KEY` / `AI_MODEL`）；`DASHSCOPE_API_KEY`、`DEEPSEEK_API_KEY` 仅作为直连供应商时的备用兼容项，默认留空
2. **知识库构建**：
   - 将所有产品信息（名称/品牌/价格/规格/描述）导出为文本文档
   - 将常见问题（配送范围/支付方式/退换货政策等）编写为 FAQ 文档
   - 使用向量数据库（可用 Prisma + pgvector 或简单的关键词匹配作为 v1）
3. **RAG 流程**：用户提问 → 检索相关知识 → 拼接 System Prompt + 知识 + 用户问题 → 大模型生成
4. **System Prompt 要求**：
   ```
   你是华启商城的AI客服助手"小启"，专门服务湘潭地区的客户。
   你可以回答关于产品信息、价格、配送、支付等问题。
   回答要简洁友好，使用口语化中文。
   不确定的问题请建议用户联系人工客服。
   ```
5. **聊天界面** `/shop/ai-chat`：
   - 气泡式对话 UI（用户右侧蓝色，AI左侧灰色）
   - 流式输出（SSE 或 fetch streaming）
   - 快捷问题标签（"有什么推荐？"/"怎么下单？"/"配送多久？"）
   - 对话历史存储至 ChatHistory 表
6. **API Route**：`/api/ai/chat` — POST 接口，SSE 流式响应

### 验收标准
| # | 验收项 | 通过条件 |
|---|--------|---------|
| 1 | 产品问答 | 问"茅台多少钱"能返回正确价格 |
| 2 | 流式输出 | 回答逐字显示，非等待全部生成后一次性显示 |
| 3 | 知识边界 | 问无关问题（如"明天天气"）回复"建议联系人工客服" |
| 4 | 对话历史 | 刷新后能加载之前的对话记录 |
| 5 | 快捷问题 | 点击快捷标签自动发送对应问题 |
| 6 | 错误处理 | API 超时/异常时显示友好错误提示 |

## Step 4.2 用户画像系统（Day 14）

### 开发说明
1. **标签体系**（存储在 UserProfile.tags JSONB 字段）：
   - 消费能力：月均消费 >500 → HIGH，100-500 → MEDIUM，<100 → LOW
   - 品类偏好：按订单中各品类金额占比，取 TOP2
   - 购买频次：月均 ≥4次 → HIGH，1-3次 → MEDIUM，<1次 → LOW
   - 生命周期：注册 <30天 → NEW，最近30天有购买 → ACTIVE，30-90天无购买 → SILENT，>90天 → LOST
2. **自动打标引擎** `src/features/ai/profile-engine.ts`：
   - 定时任务或按需触发（每次订单完成后更新该客户画像）
   - 查询客户的历史订单数据
   - 按规则计算标签并更新 UserProfile
3. **后台画像展示** — 客户详情页增加画像卡片：
   - 标签展示（彩色 Tag）
   - 消费趋势折线图（近 6 个月）
   - 最常购买的 TOP5 产品

### 验收标准
| # | 验收项 | 通过条件 |
|---|--------|---------|
| 1 | 自动打标 | 有5笔以上订单的客户能自动生成正确的标签 |
| 2 | 标签准确 | 月均消费600元的客户标记为 HIGH |
| 3 | 品类偏好 | 主要买白酒的客户偏好标签包含"白酒" |
| 4 | 生命周期 | 超90天未购买的客户标记为 LOST |
| 5 | 画像展示 | 客户详情页正确展示标签和消费趋势图 |

## Step 4.3 趣味互动模块（Day 15）

### 开发说明
1. 路由：`src/app/(shop)/fun/page.tsx`
2. **节气养生推荐**：
   - 根据当前日期判断节气（二十四节气算法或查表）
   - 调用 AI 生成当前节气的养生建议（缓存结果，每节气生成一次）
   - 关联推荐：春季→推荐果酒/花茶；冬季→推荐白酒/暖饮
   - 页面：节气名称 + 插图(AI生成) + 养生文案 + 推荐产品卡片
3. **AI 性格测试"你是什么酒的性格"**：
   - 5~8 道选择题（如"朋友聚会你更喜欢？A.热闹 B.小酌"）
   - 根据选择 AI 生成个性化结果（"你是酱香型白酒性格：沉稳内敛..."）
   - 结果页：酒的图片 + 性格描述 + 推荐产品 + 分享按钮
   - 用户作答时自然采集偏好数据，写入 UserProfile
4. **每日签到**：
   - 日历式签到组件
   - 连续签到奖励积分（1日5分, 7日50分, 30日200分）
   - 积分后续可兑换优惠券

### 验收标准
| # | 验收项 | 通过条件 |
|---|--------|---------|
| 1 | 节气正确 | 页面显示的节气与当前日期对应的节气一致 |
| 2 | 产品关联 | 推荐的产品与当前季节/节气相关 |
| 3 | 测试流程 | 5道题全部作答后显示个性化结果 |
| 4 | 画像采集 | 完成测试后用户的偏好标签自动更新 |
| 5 | 分享功能 | 测试结果页有分享按钮（生成分享图片或链接） |
| 6 | 签到功能 | 每日只能签到一次，连续签到积分正确累加 |

## Step 4.4 精准营销（Day 16）

### 开发说明
1. **个性化推荐**：商城首页"猜你喜欢"区块
   - 基于用户画像的品类偏好推荐
   - 未登录/新用户展示热门商品
2. **优惠券系统**：
   - 后台创建优惠券：名称 / 类型(满减AMOUNT/折扣PERCENT) / 面额 / 使用门槛 / 有效期 / 发放数量
   - 定向发放：按用户标签筛选目标用户群 → 批量发放
   - 用户端：我的优惠券列表（未使用/已使用/已过期）
   - 下单时选择优惠券，自动计算折扣
3. **运营仪表盘** `/dashboard` 增加运营指标 Tab：
   - 用户增长曲线（日新增注册）
   - 用户活跃度（DAU/MAU）
   - 标签分布饼图（消费能力/生命周期）

### 验收标准
| # | 验收项 | 通过条件 |
|---|--------|---------|
| 1 | 推荐相关性 | 爱买白酒的用户首页推荐白酒相关产品 |
| 2 | 优惠券创建 | 后台可创建满减/折扣类型优惠券 |
| 3 | 定向发放 | 可按"HIGH消费能力"标签筛选用户并批量发券 |
| 4 | 下单使用 | 结算时选择优惠券后金额正确减免 |
| 5 | 券规则校验 | 不满足使用门槛的优惠券灰显不可选 |
| 6 | 运营数据 | 仪表盘数据与数据库实际数据一致 |
