## Step 3.2 智能分单系统（Day 10）

### 开发说明

**核心分单算法** `src/features/orders/routing.ts`：

```
输入：Order（含 OrderItems）
处理逻辑：
1. 遍历每个 OrderItem
2. 查询该产品的 bulkThreshold
3. 如果 item.quantity >= bulkThreshold → 标记为大单项
4. 否则 → 标记为散单项
5. 全部为大单项 → routingType = WAREHOUSE（总仓直发）
6. 全部为散单项 → routingType = DEALER（就近经销商）
7. 混合 → 拆分为两个子订单（父子订单关联）
8. DEALER 类型：
   a. 获取收货地址的经纬度
   b. 查询所有 isAccepting=true 的 Dealer
   c. 计算与每个 Dealer 的距离（Haversine 公式或高德API）
   d. 按距离排序，选择最近的
   e. 创建 OrderRouting（status=PENDING）
   f. 经销商拒单后自动匹配下一个最近的
```

**地图集成：**
1. `/dashboard/map` 页面集成高德地图 JS API 2.0
2. 地图上标注所有经销商位置（图标 Marker）
3. 点击 Marker 显示经销商信息弹窗
4. 订单热力图图层（根据历史订单的收货地址密度）
5. 经销商服务半径圈（Circle 覆盖层）

### 验收标准
| # | 验收项 | 通过条件 |
|---|--------|---------|
| 1 | 散单路由 | 订单所有商品 < bulkThreshold 时，匹配最近经销商 |
| 2 | 大单直发 | 订单任一商品 >= bulkThreshold 时，标记为总仓直发 |
| 3 | 混合拆分 | 包含大小单的订单自动拆分为两个关联子订单 |
| 4 | 距离计算 | 经销商匹配结果与手动计算距离一致（误差 < 1km） |
| 5 | 拒单重匹配 | 经销商拒单后自动分配给下一个最近的 |
| 6 | 地图展示 | 高德地图正确加载，经销商 Marker 位置正确 |
| 7 | 热力图 | 热力图根据订单地址密度正确渲染 |

## Step 3.5 财务对账（Day 12）

### 开发说明
1. **财务总览** `/finance`：应收总额 / 应付总额 / 本月收入 / 收入趋势图
2. **应收账款** `/finance/receivable`：
   - 按客户汇总：客户名 / 总欠款 / 最早逾期天数 / 操作(登记收款)
   - 账龄分析：30天内 / 30-60天 / 60-90天 / 90天以上 的金额占比（堆叠柱状图）
3. **收款登记** `/finance/payments`：
   - 选择客户→显示其待核销订单列表→选择订单→输入金额→选方式(现金/转账/微信)
   - 支持一次核销多张订单
   - 支持部分收款（订单部分还款）
   - 赊账订单到期提醒（dueDate < 当前日期 标红）
4. **对账单** `/finance/statements`：
   - 选择客户 + 起止日期 → 生成对账明细
   - 包含：期初余额 / 本期发生额（每笔订单和收款） / 期末余额
   - 导出 PDF（使用 html2canvas + jsPDF 或服务端生成）
   - 打印预览

### 验收标准
| # | 验收项 | 通过条件 |
|---|--------|---------|
| 1 | 应收准确 | 客户应收 = Σ已完成订单应付金额 - Σ已收款金额 |
| 2 | 账龄计算 | 按订单创建日期到今天的天数分桶，金额正确 |
| 3 | 收款核销 | 收款后对应订单的 paidAmount 更新，全额付清后订单状态标记 |
| 4 | 部分收款 | 支持订单部分付款，剩余金额正确 |
| 5 | 对账单 | 期初+本期发生-本期收款=期末，数据一致 |
| 6 | PDF导出 | 导出的 PDF 内容完整，格式整齐 |
| 7 | 逾期预警 | 超过30天账期的赊账订单标红显示 |
