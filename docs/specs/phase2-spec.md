## Step 2.1 商城布局（Day 5 上午）

### 开发说明
1. `src/app/(shop)/layout.tsx` — 商城主布局，移动端优先
2. **顶部栏**：Logo(左) + 搜索框(中) + 登录/头像(右)
3. **底部 TabBar**（仅移动端可见，固定底部）：
   - 首页(Home图标) / 分类(Grid图标) / 购物车(ShoppingCart图标+角标数量) / 我的(User图标)
   - 当前 Tab 高亮（红色主色 `#dc2626`）
4. PC端（≥768px）：顶部导航替代 TabBar，分类横向展示
5. 整体配色：暖白背景 `#fafaf9`，主色中国红 `#dc2626`
6. 适老化开关：存储在 localStorage，切换后字体放大 25%

## Step 2.2 商城首页（Day 5）

### 开发说明
1. 路由：`src/app/(shop)/page.tsx`（Server Component）
2. **Banner 轮播**：使用 Embla Carousel 或自行实现，3~5 张 Banner 自动轮播（4s 间隔）
3. **分类入口**：酒类/食品/饮料 三个图标卡片，圆形图标+文字，点击跳转分类页
4. **热门推荐**：横向滚动卡片列表（产品图+名称+价格），取 salesCount TOP 8
5. **新品上架**：2列网格（移动端）/ 4列（PC），取最近上架8个产品
6. 产品卡片：图片（占位图） + 名称（单行省略） + 价格（红色¥XX.XX）+ 加购按钮
7. 所有价格显示 retailPrice

## Step 2.3 产品目录（Day 5~6）

### 开发说明
1. 路由：`src/app/(shop)/catalog/page.tsx`，支持 query 参数 `?category=xxx&brand=xxx&sort=xxx&q=xxx`
2. 顶部分类 Tab（酒类/食品/饮料/全部），选中后筛选
3. 搜索框：URL 参数驱动，输入后防抖 500ms 更新 URL
4. 筛选：品牌多选 / 价格区间滑块
5. 排序：综合(默认) / 价格升/降 / 销量 / 新品
6. 产品网格 + 无限滚动（使用 Intersection Observer，每次加载20条）
7. 空结果提示"未找到相关商品"

## Step 2.4 产品详情页（Day 6）

### 开发说明
1. 路由：`src/app/(shop)/product/[id]/page.tsx`
2. 图片区：轮播（多图时可滑动）
3. 信息区：名称 / 价格(大红色) / 规格 / 品牌 / 库存状态(有货/缺货)
4. 数量选择器：- / 数量 / +，最小1，最大不超过库存
5. 操作按钮：加入购物车(橙色) + 立即购买(红色)

## Step 2.6 下单与支付（Day 7）

### 开发说明
1. 路由：`src/app/(shop)/checkout/page.tsx`（需登录）
2. **收货地址**：显示默认地址，可切换/新增，新增时限制城市为"湘潭市"
3. **商品清单**：从购物车选中项获取，显示列表
4. **支付方式**：第一版先做"模拟支付"（按钮点击即完成支付）
5. **提交订单** Server Action 逻辑：
   - 校验库存充足
   - 创建 Order + OrderItem 记录
   - 扣减库存（事务操作）
   - 清除购物车已购商品
   - 生成唯一订单号（格式：HQ + 年月日 + 6位序号，如 HQ20260425000001）
   - 触发分单逻辑（Phase 3 实现，此阶段先标记为 WAREHOUSE）
6. **支付成功页**：显示订单号 + 动画 + "查看订单"/"继续购物"按钮

### 验收标准
| # | 验收项 | 通过条件 |
|---|--------|---------|
| 1 | 地址限制 | 新增地址时城市只能选"湘潭市" |
| 2 | 库存扣减 | 下单后产品库存正确减少，StockRecord 有出库记录 |
| 3 | 订单生成 | Order 和 OrderItem 数据正确，金额计算无误 |
| 4 | 订单号格式 | 格式为 HQYYYYMMDDXXXXXX，不重复 |
| 5 | 购物车清除 | 下单成功后购物车中已购商品被移除 |
| 6 | 事务安全 | 库存不足时整个下单回滚，不生成脏数据 |
| 7 | 成功页 | 显示订单号和成功提示 |
