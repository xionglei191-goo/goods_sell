# Phase 5：微信生态 + 优化 — 开发规格书

> 负责人：代码组 | 审核人：架构师 | 工期：后续迭代

## Step 5.1 微信小程序

### 开发说明
1. 技术选型：Taro 4.x（React 语法，可复用商城组件逻辑）
2. 复用 Phase 2 的商城核心逻辑，适配小程序 API
3. 微信登录（wx.login → 后端 code2session）
4. 微信支付（小程序支付 JSAPI）
5. 分享：自定义分享卡片（标题+图片+路径）

### 验收标准
| # | 验收项 | 通过条件 |
|---|--------|---------|
| 1 | 小程序审核 | 提交微信审核通过 |
| 2 | 微信登录 | 一键授权登录，自动关联或创建账号 |
| 3 | 微信支付 | 小程序内支付流程完整，回调处理正确 |
| 4 | 分享裂变 | 分享到群聊/朋友圈的卡片可正常打开 |
