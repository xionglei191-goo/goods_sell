# 图片素材采集辅助工具

这个工具用于临时补充商品展示图。它不会自动爬取京东或其他电商详情页，只处理人工确认后的图片 URL。

## 后台管理

后台已接入图片素材管理页面：

```text
产品管理 -> 图片素材
/dashboard/products/materials
```

后台页面支持：

- 按商品登记图片 URL 或直接上传图片。
- 记录来源页面、来源名称、授权状态和备注。
- 按缺主图、审核状态、授权状态筛选商品素材。
- 下载 CSV 模板，批量预检并导入商品素材。
- 批量通过并设主图、批量拒绝、批量删除。
- 展示图片格式、大小、尺寸、Hash 和重复素材提示。
- 记录授权附件 URL，便于后续追溯。
- 审核通过后将素材写入 `ProductImage` 并设为商城主图。
- 拒绝或删除素材记录，并写入操作日志。

素材来源和审核记录保存在 `ProductImageMaterial` 表；商城实际展示图仍使用 `ProductImage` 表。

### CSV 批量导入字段

后台 CSV 导入支持以下字段：

| 字段 | 说明 |
|------|------|
| sku | 商品 SKU，必填 |
| candidateImageUrl | 图片 URL，支持 http(s) 或 `/images/products/...` |
| sourcePage | 来源页面 |
| sourceName | 来源名称 |
| licenseStatus | 授权状态 |
| approved | 只有 `TRUE` 才会导入 |
| authAttachmentUrl | 授权附件 URL |
| notes | 备注 |

### 图片存储配置

默认使用本地存储：

```bash
IMAGE_MATERIAL_STORAGE_PROVIDER="LOCAL"
```

如需切换 Cloudflare R2：

```bash
IMAGE_MATERIAL_STORAGE_PROVIDER="CLOUDFLARE_R2"
R2_ACCOUNT_ID=""
R2_ACCESS_KEY_ID=""
R2_SECRET_ACCESS_KEY=""
R2_BUCKET=""
R2_PUBLIC_BASE_URL="https://your-cdn-domain"
```

如需切换阿里云 OSS：

```bash
IMAGE_MATERIAL_STORAGE_PROVIDER="ALIYUN_OSS"
OSS_ACCESS_KEY_ID=""
OSS_ACCESS_KEY_SECRET=""
OSS_BUCKET=""
OSS_REGION="oss-cn-hangzhou"
OSS_PUBLIC_BASE_URL="https://your-cdn-domain"
```

远程 URL 素材在审核通过并设为主图时会下载并写入当前配置的存储；上传图片会立即写入当前配置的存储。

## 使用流程

如果本地没有配置 `DATABASE_URL`，请在服务器部署目录执行：

```bash
ssh goods-sell-vps
cd /data/goods_sell
```

1. 导出商品素材模板：

```bash
npm run images:template
```

默认生成：

```text
image-materials/product-image-candidates.csv
```

2. 人工填写 CSV：

| 字段 | 说明 |
|------|------|
| sku | 商品 SKU，自动导出，不要修改 |
| candidateImageUrl | 你确认可用的图片直链 |
| sourcePage | 图片来源页面，便于追溯 |
| sourceName | 来源名称，例如品牌方素材包、供应商、自拍 |
| licenseStatus | 授权状态 |
| approved | 只有 `TRUE` 才会导入 |
| notes | 备注 |

`licenseStatus` 允许值：

```text
authorized
brand-provided
supplier-provided
owned
public-domain
cc
internal-demo-approved
```

3. 先 dry-run 检查：

```bash
npm run images:import -- --file image-materials/product-image-candidates.csv
```

4. 确认无误后正式导入：

```bash
npm run images:import -- --file image-materials/product-image-candidates.csv --apply
```

导入后会：

- 下载图片到 `public/images/products`
- 将该图设为商品主图
- 把来源记录写入 `image-materials/import-log.jsonl`

## 注意事项

- 不要把未授权的电商平台商品图用于正式公网商城。
- 京东、天猫、淘宝等平台的商品图通常涉及品牌方、商家或平台版权。
- 建议优先使用品牌方/供应商授权图片包，或自行拍摄商品图。
- `image-materials/*.csv` 和 `image-materials/*.jsonl` 默认不提交到 Git。
