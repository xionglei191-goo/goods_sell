# 图片素材采集辅助工具

这个工具用于临时补充商品展示图。它不会自动爬取京东或其他电商详情页，只处理人工确认后的图片 URL。

## 使用流程

如果本地没有配置 `DATABASE_URL`，请在服务器部署目录执行：

```bash
ssh goods-sell-vps
cd /data/goods_sell
```

1. 导出商品素材模板：

```bash
npm run images:template
```

默认生成：

```text
image-materials/product-image-candidates.csv
```

2. 人工填写 CSV：

| 字段 | 说明 |
|------|------|
| sku | 商品 SKU，自动导出，不要修改 |
| candidateImageUrl | 你确认可用的图片直链 |
| sourcePage | 图片来源页面，便于追溯 |
| sourceName | 来源名称，例如品牌方素材包、供应商、自拍 |
| licenseStatus | 授权状态 |
| approved | 只有 `TRUE` 才会导入 |
| notes | 备注 |

`licenseStatus` 允许值：

```text
authorized
brand-provided
supplier-provided
owned
public-domain
cc
internal-demo-approved
```

3. 先 dry-run 检查：

```bash
npm run images:import -- --file image-materials/product-image-candidates.csv
```

4. 确认无误后正式导入：

```bash
npm run images:import -- --file image-materials/product-image-candidates.csv --apply
```

导入后会：

- 下载图片到 `public/images/products`
- 将该图设为商品主图
- 把来源记录写入 `image-materials/import-log.jsonl`

## 注意事项

- 不要把未授权的电商平台商品图用于正式公网商城。
- 京东、天猫、淘宝等平台的商品图通常涉及品牌方、商家或平台版权。
- 建议优先使用品牌方/供应商授权图片包，或自行拍摄商品图。
- `image-materials/*.csv` 和 `image-materials/*.jsonl` 默认不提交到 Git。
