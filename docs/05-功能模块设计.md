## 模块八：角色权限与账号审核

| 页面/能力 | 路由 | 功能描述 |
|------|------|---------|
| 无权限页 | /forbidden | 已登录但无权限时展示当前角色、原因和可进入入口 |
| 角色化注册 | /register | 支持选择消费者注册或经销商申请；消费者即时启用，经销商进入待审核 |
| 经销商申请审核 | /dashboard/dealers | 管理员查看待审核申请，补齐门店档案后通过或驳回 |
| 角色化菜单 | 后台布局 | 按 ADMIN/SALESPERSON/WAREHOUSE/FINANCE 过滤侧边栏和子菜单 |
| 服务端权限守卫 | Server Actions/API | 库存、采购、财务、设置、日志、经销商审核等操作按岗位校验 |
| 销售员数据隔离 | 订单/客户/渠道经营页面 | 销售员默认只看名下或分派给自己的数据 |

权限体验原则：
- 菜单不可见不等于安全边界，所有写操作都必须在服务端二次校验。
- 财务、仓管、销售员可访问同一订单/客户信息时，页面行为按只读或岗位动作区分。
- 经销商和消费者不进入后台 `/dashboard`，分别使用 `/dealer` 和 `/shop` 体系。

## 模块九：AI Tools 与代办助手

### 9.1 全端 AI 悬浮气泡

| 入口 | 路由范围 | 功能描述 |
|------|----------|----------|
| 商城 AI 气泡 | `/shop/*` | 客户用自然语言查商品、查订单、查欠款、生成下单草稿和确认下单 |
| 后台 AI 气泡 | `/dashboard/*` | 老板和员工查询经营数据、客户、订单、库存、财务，发起权限内管理操作 |
| 经销商 AI 气泡 | `/dealer/*` | 经销商查询待接单、结算，发起库存上报、接单和拒单 |

气泡展示形式：

- 对话消息：承接自然语言输入和 AI 简短解释。
- 摘要卡：展示查询结果，如商品价格、库存、业绩、应收和上线检查状态。
- 确认卡：展示写操作的对象、金额、数量、风险等级和确认按钮。
- 高风险确认：要求输入指定文字后才能执行。

### 9.2 Tool 执行层

| 层级 | 文件 | 职责 |
|------|------|------|
| 运行时上下文 | `src/features/ai/tools/context.ts` | 识别当前登录用户、角色、账号类型和数据范围 |
| 固定词条 | `src/features/ai/intent-templates.ts` | 维护角色固定词条、本地模板参数和高置信固定命中 |
| Tool 注册表 | `src/features/ai/tools/registry.ts` | 声明 tool 名称、描述、schema、权限、风险等级和 handler |
| Planner | `src/features/ai/tools/planner.ts` | 将自然语言转换为 tool plan，保守处理本地规则，复杂问法交给真实 AI provider |
| Executor | `src/features/ai/tools/executor.ts` | 参数校验、权限校验、确认卡、handler 执行和错误归一 |
| Audit | `src/features/ai/tools/audit.ts` | 记录 AI 调用、确认状态、结果摘要和失败原因，敏感字段脱敏 |
| 高频候选 | `src/features/ai/prompt-usage.ts` | 记录非固定问法、命中来源、tool 计数和候选状态，供人工审核沉淀 |

AI 规划链路：

1. 前端通过 `/api/ai/quick-prompts` 获取当前角色固定词条。
2. 用户点击固定词条时，服务端优先用 `intent-templates.ts` 生成本地 tool plan。
3. 用户自由输入时，`planner.ts` 只处理高置信本地规则；含糊或复杂问法调用真实 AI provider。
4. 模型返回的 JSON plan 仍需通过 `validateAiToolPlan()` 和 executor 权限/参数校验。
5. 非固定问法只记录为高频候选，不自动启用本地规则。

Tool 风险等级：

| 等级 | 行为 |
|------|------|
| READ | 查询类，权限通过后直接执行 |
| DRAFT | 只生成草稿，不改变关键业务状态 |
| WRITE | 生成确认卡，用户确认后执行 |
| HIGH_RISK | 二次确认后执行，必须写审计日志 |

### 9.3 已覆盖业务能力

| 分组 | 能力 |
|------|------|
| 客户服务 | 商品搜索、客户默认信息、下单草稿/提交、我的订单、我的应收 |
| 经营查询 | 老板总览、销售员业绩、客户查询、客户购买历史、商品经营、财务、配送、渠道经营 |
| 管理员操作 | 商品调价/上下架/安全库存、订单状态、入库出库、盘点、客户资料、经销商审核/政策、优惠券、新品推送、员工账号、系统参数 |
| 经销商操作 | 待接单、接单、拒单、库存上报、结算查询 |
| 上线检查 | 数据库、认证、AI、地图、微信、票税、酒类资质配置检查 |

### 9.4 权限与审计原则

- ADMIN 可使用全部后台 tools。
- SALESPERSON、WAREHOUSE、FINANCE 只暴露岗位权限内 tools。
- CONSUMER 只能操作自己的商品咨询、订单、地址和账务信息。
- DEALER 只能操作自己的接单、库存、结算、推广和线索信息。
- 固定词条也按角色过滤，不把后台经营、经销商接单、财务摘要等入口统一展示给所有人。
- “某客户买了什么/购买记录”只允许 ADMIN、SALESPERSON、FINANCE 在权限范围内查询，WAREHOUSE、DEALER、CONSUMER 不可越权查看他人购买历史。
- 所有 WRITE/HIGH_RISK tool 调用写入 `AuditLog`。
- AI 回复中不展示越权数据，越权请求统一返回无权限提示。
