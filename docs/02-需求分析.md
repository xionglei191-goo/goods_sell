# 华启酒饮数字渠道平台 — 需求分析

## 0. 战略定位修正

平台定位为“区域酒水饮料总代的数字化获客与渠道协同系统”，而不是单纯 B2C 商城或对外售卖的 SaaS。业务优先级调整为：

1. 帮公司梳理商品、库存、经销商、业务员和客户线索。
2. 帮业务员地推沉淀客户和经销商绑定关系。
3. 帮下游经销商承接附近小单，降低渠道抵触。
4. 帮公司识别大单、团购、宴席和新品推广机会。
5. 用 AI 互动采集低压力需求，沉淀客户画像，再做精准推送。

## 3. 核心业务流程

### 3.1 客户接入与分流流程
```
业务员地推/经销商二维码/小程序/公众号/搜索
→ 选择身份或场景(宴席、团购、门店补货、普通购买)
→ AI互动或询价表单采集需求
→ 生成 Lead 线索和用户画像
→ 判断小单/中单/大单/门店补货/企业团购
→ 小单分给附近经销商
→ 中单生成询价，由业务员或经销商确认
→ 大单/团购转公司或指定业务员报价
→ 成交后转订单、配送、对账、复购/新品推送
```

### 3.2 订单状态流转
```
待支付 → 已支付/待确认 → 已确认/待发货 → 配送中 → 已完成
                                              ↘ 已取消
                                              ↘ 退款中 → 已退款
```

### 3.3 智能分单逻辑
- 每个产品独立设置散单/大单阈值，订单金额也参与判断
- 小单：地图API计算距离 → 匹配服务半径内、库存充足、品牌权限满足的经销商 → 推送订单
- 中单：生成询价或待确认单，避免价格体系混乱
- 大单/团购：总仓或指定业务员处理，先报价再转正式订单
- 经销商拒单 → 自动匹配下一家
- 异常低价、跨区履约、频繁拒单、客户投诉进入渠道冲突台账

### 3.4 AI互动获客逻辑

| 场景 | 采集信息 | 后续动作 |
|------|----------|----------|
| 宴席配酒助手 | 桌数、预算、品牌偏好、饮料搭配、配送时间 | 生成大单线索，转业务员报价 |
| 企业福利/送礼推荐 | 预算、收礼对象、包装、开票、配送批次 | 生成团购询价和企业客户画像 |
| 门店补货助手 | 门店类型、畅销品类、周转速度、价格敏感度 | 推荐补货组合，绑定业务员或经销商 |
| 新品试饮/趣味测评 | 口味偏好、场景、地理位置、联系方式 | 打标签，进入新品推送池 |
