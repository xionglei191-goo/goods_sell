## 3. 核心业务流程

### 3.1 C端消费者下单流程
```
浏览商城 → 加购物车 → 确认订单 → 填写地址(限湘潭) → 微信支付 → 订单生成
→ 系统判断大单/散单 → 散单匹配就近经销商 / 大单总仓直发
→ 经销商确认接单 → 配送 → 确认收货 → 完成
```

### 3.2 订单状态流转
```
待支付 → 已支付/待确认 → 已确认/待发货 → 配送中 → 已完成
                                              ↘ 已取消
                                              ↘ 退款中 → 已退款
```

### 3.3 智能分单逻辑
- 每个产品独立设置散单/大单阈值
- 散单：地图API计算距离 → 匹配最近有库存经销商 → 推送订单
- 大单/团购：总仓直接配送
- 经销商拒单 → 自动匹配下一家

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 首屏 <2s，API <500ms，500并发 |
| 安全 | HTTPS、bcrypt、SQL注入防护、XSS/CSRF防护 |
| 兼容 | Chrome/Safari/Edge/Firefox + 微信浏览器 |
| 响应式 | 移动优先，适老化大字体选项 |
| 可维护 | TypeScript、模块化、操作日志、DB迁移版本化 |
